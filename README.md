HobbyMatch: Dating App для поиска партнеров по хобби
Сервис для поиска единомышленников и партнеров для совместных занятий спортом, играми или творчеством. Проект реализует сложную многоуровневую систему ранжирования пользователей.

1. Технологический стек
  Language: Python 3.10+
  Framework: Aiogram 3.x (Bot), FastAPI (Backend)
  Database: PostgreSQL (Основное хранилище)
  Cache: Redis (Кэширование анкет и состояний)
  Message Broker: RabbitMQ (Асинхронная обработка лайков и рейтингов)
  Task Queue: Celery (Регулярные пересчеты рейтингов)
  Storage: Minio / S3 (Хранение фото анкет)

2. Архитектура микросервисов (Microservices Description)
Система разделена на независимые функциональные блоки, развернутые в отдельных Docker-контейнерах:

  a. Gateway & Bot Service (Aiogram 3.x)
    Назначение: Единственная точка входа для пользователей через Telegram API.
    Функционал: Обработка команд (/start, /profile), реализация пошаговой регистрации (FSM), отображение карточек анкет.
    Взаимодействие: Запрашивает данные у Core API по HTTP/REST. Отправляет "сырые" события действий пользователя в очередь RabbitMQ.
   
  b. Core API Service (FastAPI)
    Назначение: Сервис бизнес-логики и управления данными.
    Функционал: CRUD операции с анкетами, хранение медиа-ссылок, фильтрация пользователей по интересам и городу.
    Взаимодействие: Работает напрямую с PostgreSQL. Использует Redis для кэширования "пачек" анкет (по 10 штук), чтобы снизить нагрузку на          основную БД.
    
  c. Scoring & Analytics Worker (Celery / RabbitMQ Consumer)
    Назначение: Фоновый процессинг тяжелых вычислений.
    Функционал: Слушает очередь событий (лайки/дизлайки). На основе этих данных обновляет behavior_score (Уровень 2 рейтинга).
    Взаимодействие: Асинхронно обновляет таблицу ratings в PostgreSQL. Не блокирует работу бота при расчетах.
    
  d. Infrastructure Services
    PostgreSQL: Хранилище профилей, истории мэтчей и системных логов.
    Redis: * Кэш пред-отранжированных списков анкет для мгновенной отдачи пользователю.
    Хранилище состояний бота (FSM Storage).
    RabbitMQ: Шина данных для обмена сообщениями между Ботом и Скорринг-воркером.
    Minio (S3): Хранение фотографий снаряжения и аватарок пользователей.

4. Алгоритм ранжирования (Ranking System)
Реализовано три уровня оценки анкет:
  Уровень 1 (Первичный): Совпадение типа хобби, географическая близость и разница в уровне навыка (skill_level).
  Уровень 2 (Поведенческий): Динамический вес на основе соотношения лайков/пропусков и частоты мэтчей.
  Уровень 3 (Комбинированный): Интеграция всех метрик с бонусными коэффициентами (например, за наличие своего снаряжения).
